Git 冲突解决方案

发表于 2017-12-31 分类于 Git 阅读次数：
本文字数： 2.1k 阅读时长 ≈ 4 分钟

说明

在团队协作使用Git时，遇到的冲突通常为两种，分别为：

  • git pull 拉取冲突，本地修改的文件与远程仓库待拉取的文件内容冲突。
  • git push 推送冲突，本地仓库提交的版本基于远程仓库的旧版本。

拉取冲突

遇到这种情况，你可以：

Checkout 丢弃

丢弃本地中发生冲突的修改，然后再拉取。

Stash 贮藏

贮藏修改，然后拉取，接着恢复贮藏，解决冲突。

编辑冲突文件，会看到如下内容：

在<<<<<<< Updated upstream和=======之间的是拉取下来的内容；
在=======和>>>>>>> Stashed changes之间的是本地修改的内容。
根据需要丢弃无用内容，保留有用的内容。

推送冲突

同一个分支上有多个人维护时，建议在commit前进行pull操作。
如果未pull就进行commit、push操作，很有可能造成版本冲突，从而产生不必要的合并提交。

这种情况是因为远程仓库的分支比本地的新，有以下方法解决这种情况：

Merge 合并
